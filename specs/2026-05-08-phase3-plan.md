## 0. Plan 사용 규칙

- 각 task는 **단일 commit 단위**. 커밋 prefix `WI-T<NN>-<type>(<scope>): 한글 작업명`.
- 1 commit = 1 task. fixer commit이 필요하면 `WI-T<NN>.5-fix(<scope>): ...`로 별도.
- 구현 전 **AC (Acceptance Criteria)** 충족 여부 명시. 검증 명령은 task별로 inline 명시.
- **TDD + cumulative project lock 유지**: 신규 source file은 mypy --strict 통과해야 다음 task 진입.
- Phase 3-A 종료 시점에 **3-reviewer audit** (R1 보안 / R2 아키 / R3 정직성) 병렬.

### T29 — `cases/_protocols.py` ScenarioResult TypedDict + Backends dataclass 설계 골격

- **변경 파일**: `cases/_protocols.py` (신규)
- **내용**:
  - `ScenarioResult` TypedDict (case_id, summary_text, output_files, metrics, failures)
  - `Backends` frozen dataclass + `OCRBackend`/`AIBackend`/`MessagingBackend` Protocol
  - 아직 case scenario는 변경 안 함 — 타입 정의 파일만
- **AC**:
  - mypy --strict 통과 (cumulative lock 53 → 54 source files)
  - `from cases._protocols import ScenarioResult, Backends` 가능
- **검증**:
  ```bash
  uv run mypy --strict core runner.py cases
  uv run ruff check . && uv run ruff format --check .
  ```

### T30 — `core/backends.py` 기본 구현 + 단위 테스트

- **변경 파일**: `core/backends.py` (신규), `tests/test_backends.py` (신규)
- **내용**:
  - `MLXOCRBackend` (현 `core.ocr.gemma.extract` 위임)
  - `OpenRouterAIBackend` (현 `core.ai.client.chat` 위임)
  - `DiscordWebhookBackend`, `GmailBackend` (현 messaging 모듈 위임)
  - `SafeBackend` (deterministic dummy — monkey patch 의존 제거)
  - `CachedBackend(real, store: CacheStore)` (현 cache_path 로직 protocol화)
  - 팩토리: `default_backends()` → 기본 조합. `safe_backends()` → 모두 SafeBackend.
- **AC**:
  - 각 backend가 `cases._protocols.OCRBackend` 등 protocol 만족 (구조적 typing)
  - `tests/test_backends.py` 신규 30+ tests (mock 기반)
  - 기존 `tests/test_safe_mode.py` / `test_ocr_gemma.py` 회귀 0
- **검증**: `uv run pytest tests/test_backends.py tests/test_safe_mode.py tests/test_ocr_gemma.py -q`

### T31 — `core.common.safe_mode_v2` contextvars 기반 + shim

- **변경 파일**: `core/common/safe_mode_v2.py` (신규), `tests/test_safe_mode_v2.py` (신규)
- **내용**:
  - `_SAFE_VAR: ContextVar[bool]`
  - `is_safe()`, `safe_mode_scope(enabled)` context manager
  - 기존 `core.common.safe_mode.is_safe()`가 v2의 ContextVar를 위임 호출 (default를 env에서 1회 초기화)
  - 기존 `intercept(case_id, apis)` (monkey patch)는 deprecate warning + Backends DI 안내 docstring
- **AC**:
  - CLI 모드에서 `DEMO_SAFE=1` env가 ContextVar에 정상 반영
  - 동시 thread 두 개가 각자 다른 scope 사용 시 격리 확인 (테스트)
  - 기존 539 + 4 skipped 회귀 0
- **검증**: 전체 pytest 통과 + 신규 contextvars 격리 tests
