# Case 04 — 시연 대본 (미수금 단계별 Discord 알림)

페르소나: **박과장** (회수 담당). 매주 회수 체크·연락 약 2시간 → **8초**.

## 1분 버전 (고객 미팅 임팩트)

**[10초] 페인**
> "박과장님이 매주 미수금 거래처에 단계별로 다른 톤으로 연락하시잖아요. 친근 → 정중 → 단호 → 법무 — 일일이 작성하다 보면 누락도 생기고 톤 일관성도 깨집니다."

**[5초] 실행**
```bash
DEMO_SAFE=1 uv run python runner.py case04_discord_overdue_alert --safe
```

**[30초] 결과**
- 콘솔: `전송 60건 / 실패 0건` (8초 내 완료)
- Discord 채널 화면 전환:
  - 🔵 friendly 24건 (안부 + 확인)
  - 🟠 neutral 18건 (정중 독촉)
  - 🔴 strict 12건 (단호)
  - ⚫ final 6건 (법무 escalation 통지)
- 임베드 색상이 한눈에 단계 구분.

**[15초] 임팩트**
> "60건을 8초에. 단계 톤은 코드에 묶여 있으니 누가 돌려도 일관됩니다. 사내 NDA 채널에만 가니까 카카오·메일 약관 리스크도 없습니다."

## 3분 버전

1분 버전 + 다음 항목 추가:

- **입력 데이터 시연** (15초): `personas/sample_data/overdue_invoices.xlsx` Quick Look — 거래처/거래번호/금액/납기일/연체일/담당자.
- **단계 분기 코드 펼침** (30초): `classify_level(days)` 함수. "0~14, 15~30, 31~60, 60+ — 4단계가 boundary 1개 변수로 정해집니다. 회사마다 정책이 다르면 여기만 고치면 됩니다."
- **column_map 강조** (15초): "다른 회계SW에서 export한 엑셀이 컬럼명이 달라도 column_map 인자만 바꾸면 같은 시나리오가 그대로 돕니다."
- **Q&A 여유** (15초)

## 5분 버전 (강의)

3분 버전 + 다음 항목 추가:

- **회수율 비교** (60초): 수기 회수 vs 자동 알림 도입 후 회수율 변화. 단계별 연락 누락 0% → 회수 사이클 단축.
- **고객사 적용 가능성** (30초):
  - 사내 NDA 채널 분리 — 거래처명·금액 노출 정책 명시.
  - 공개 webhook 사용 금지, secrets는 1Password/age 외부 보관.
  - 카카오톡 자동발송 약관 리스크 회피 사례.
- **일반화 패턴** (30초): "단계별 톤 분기"는 미수금뿐 아니라 SLA 위반 알림, 재고 부족 단계, 근태 결근 누적에도 동일 적용. 도메인만 바꾸면 됩니다.
- **단일 patch point 아키텍처** (30초): `discord.send`만 가로채면 라이브/safe 모드 전환이 무료. `send_with_level`은 그 위 레이어. 시연 환경 안정성의 비밀.
