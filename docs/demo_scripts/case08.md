# Case 08 — 시연 대본 (세금계산서 OCR → 회계 CSV)

박과장 페르소나. AX상사 관리부 박과장이 매월 거래처 세금계산서 30장을 회계SW에 수기 입력하면서 양측 사업자번호 오타를 일일이 검수하는 4시간 야근을 끝내는 시연.

## 1분 버전 (고객 미팅 도입부)

**[10초] 페인 제시**
> "AX상사 박과장님이 매월 세금계산서 30장 회계SW 입력하는 데 4시간. 더 짜증 나는 건 사업자번호 한 자리 오타 났는데 막상 알게 되는 시점은 부가세 신고할 때라는 거죠. 거기서 다시 거꾸로 추적하면 또 반나절."

**[5초] 실행 (5장만)**
```bash
uv run python -m cases.case08_ocr_invoice_to_csv.scenario
# input/에 5장만 두면 30초 처리
```
→ 콘솔에 timer: `세금계산서 OCR (5장) 완료: 30.4s`

**[30초] 결과**
- Quick Look으로 invoices_utf8.csv 열기 (Excel이 BOM 자동 인식 — 한글 깨짐 0)
- "거래일·거래번호·공급자번호·공급자명·공급받는자번호·공급받는자명·공급가액·부가세·합계 9컬럼. 세금계산서 표준 그대로."

**[15초] 임팩트**
> "Gemma 4 E4B는 로컬 Ollama. 세금계산서가 외부 클라우드로 안 나갑니다. 그리고 사업자번호 체크섬 검증이 자동이라 '오타로 잘못 친 비번호' 자체가 발생할 수 없습니다 — 검증 실패 항목은 따로 빼서 워크큐로 빠집니다."

## 3분 버전 (메인 시연)
1분 버전 + 다음 추가:

**[+30초] 30장 일괄 시연**
- input/ 비우고 `personas/sample_data/invoices_scanned/` 30장 폴더 사용
- 처리 중 콘솔 출력 시연: 세금계산서당 ~6초, 합산 ~3분
- `[OCR] inv_007.png → INV-2026-00007 / 공급자 (주)... / 공급가액 350,000원 (면세)`
- "면세 거래는 vat=0으로 자동 인식. validate_biznum + vat 일치성 검증을 둘 다 통과한 28건만 verified, 의도적으로 깨놓은 2건은 validation_failures.json에 분리."

**[+45초] dual encoding 시연 (utf-8 BOM vs cp949)**
- `invoices_utf8.csv` Excel로 열기 — 한글 정상 + BOM 자동 인식
- `invoices_cp949.csv` 회계SW(가상 더존) import 경로로 열기 — cp949 없이는 한글 깨짐 발생하는 레거시 환경 시연
- "회계SW가 어떤 인코딩 받든 그 자리에서 import 가능. 둘 중 하나만 export하면 시연 현장에서 '저희 환경은 다릅니다' 들으면 미팅이 끊겨요."

**[+15초] Q&A 핵심**
- "사업자번호 검증은 어떻게?" → "국세청 모듈러스 알고리즘. OCR이 한 자리 잘못 읽으면 체크섬에서 즉시 fail."
- "면세 거래는?" → "vat=0 케이스를 게이트에서 통과시키도록 명시 분기 — 면세사업자 거래도 정상 처리."
- "case07과 차이?" → "E2B는 영수증 빠르게 (~600ms), E4B는 세금계산서처럼 양식이 큰 거 정확하게 (~6s). 모델 선택 = 처리시간 vs 정확도 트레이드오프."

## 5분 버전 (강의)
3분 버전 + 다음 추가:

**[+1분] 회계SW dual import 시연**
- 결과 utf-8 csv → 최신 회계SW (utf-8 사용) 거래입력 import
- 결과 cp949 csv → 레거시 ERP (cp949 only) 거래입력 import
- 9컬럼이 양쪽 다 정확히 매핑되는 화면 시연

**[+1분] 가상 데이터 한계 명시 (정직성)**
> "여기 시연한 30장은 Pillow로 합성한 가상 세금계산서입니다. 사업자번호도 알고리즘으로 체크섬 만족하는 임의값이지 실 사업자등록번호가 아닙니다. 의도적으로 2장은 사업자번호 체크섬을 깨놨고요 — 검증 게이트가 실제로 거른다는 걸 보여드리려고. 도입 환경에선 사용 환경 세금계산서 5장 정도 hold-out 검증을 권장합니다."

**[+30초] 향후 개선**
- 부가세 신고 자동 import (홈택스 연동) — Phase 3 검토. CSV export까지가 스코프, 외부 자동 제출은 미스코프.
- 도장·서명 영역 제거 → OCR 정확도 향상 (Phase 3 후속).
- 거래처별 사업자번호 마스터 매칭 → 신규 vs 기존 거래처 자동 분류.
